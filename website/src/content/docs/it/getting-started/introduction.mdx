**yRest** è un server REST API mock senza configurazione per sviluppatori frontend. Legge un file `db.yml` ed espone automaticamente una REST API CRUD completa — incluse relazioni, parametri di query, paginazione, stream SSE e route statiche personalizzate — senza bisogno di un backend.

```bash
npx @yrest/cli serve db.yml
# → REST API su http://localhost:3070
```

## Quando usarlo

- Stai costruendo un frontend e hai bisogno di un'API realistica prima che il backend sia pronto.
- Hai bisogno di un mock prevedibile per test di integrazione (Vitest, Playwright, Cypress).
- Vuoi fare una demo di un flusso UI senza mantenere un server reale.
- Hai bisogno di SSE o di un comportamento di endpoint personalizzato che `json-server` non può gestire.

## Confronto

Sia yRest che `json-server` condividono la stessa filosofia zero-config — puntali a un file di dati e ottieni subito una REST API. La differenza sta in quanto lontano arriva ciascuno. yRest estende il modello con un sistema di relazioni più ricco, operatori di query tipizzati, stream SSE, scenari mock condizionali e una API programmatica completa. Se `json-server` copre le tue esigenze, continua a usarlo. Se hai raggiunto i suoi limiti, yRest è il passo successivo.

## Installazione

Aggiungi yRest al tuo progetto come dipendenza di sviluppo:

```bash
npm install -D @yrest/cli
```

Oppure eseguilo direttamente senza installare — utile per una prova rapida:

```bash
npx @yrest/cli serve db.yml
```

Una volta installato, crea i file di avvio e lancia il server:

```bash
npx @yrest/cli init          # crea db.yml + yrest.config.yml nella directory corrente
npx @yrest/cli serve db.yml  # → REST API su http://localhost:3070
```

Il server osserva le modifiche a `db.yml` e si ricarica automaticamente. Vai all'[Avvio rapido](/it/getting-started/quick-start/) per una guida passo dopo passo con un esempio reale.
